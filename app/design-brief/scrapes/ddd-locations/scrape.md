# UX Analysis: DDD Locations
URL: https://dinersdriveinsdiveslocations.com/

## UX Patterns Extracted
- **Search implementation:** Primary navigation uses text links grouping locations "by CITY", "by STATE", "by FOODS", "by CUISINE". There are large blue call-to-action buttons for "SEARCH DINERS by STATES" and "SEARCH DINERS by CITIES". No explicit text input search on the main page.
- **Restaurant card fields:** Displayed as list items containing a hyperlinked name, an image thumbnail, street address, city (as a link), state, zip code, and phone number. Also includes episode info (name and air date) prominently.
- **Navigation structure:** Top navigation bar (City, State, Foods, Cuisine, Road Trip, Recipes, Shop). The homepage lists all 50 states and international locations below the hero section.
- **Color palette:** 
  - Primary text/accent: `rgb(218, 126, 18)` (approx `#DA7E12` orange).
  - Buttons: Blue (`#337AB7` or similar standard bootstrap primary blue based on visual impression).
  - Background: Mostly white/light gray.
- **Typography:** Uses standard sans-serif fonts for headings and body.
- **Mobile approach:** The layout breaks down into a single column for state/city lists on mobile.
- **Trivia/Fun elements:** "Road Trip" builder page, latest episode highlights.
- **Map integration:** Shows static images of maps or lists grouped by region; no interactive map visible in the primary flow.
